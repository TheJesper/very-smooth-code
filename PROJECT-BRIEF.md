# Very Smooth Code - Project Brief

> **Strategic Research:** See [Initial Research](./docs/init-research/initial-research.md) for comprehensive analysis including trademark considerations, SEO architecture, high-traffic tool categories, and monetization strategies.

## Vision

A developer hub for smart coding help services, styling, web, CSS mini tools, and curated resources. Links to cool stuff, hoffbit articles, and more.

## Brand

**Very Smooth Code** (VSC) - chosen to avoid Microsoft Visual Studio Code trademark issues.

- **Domain**: vscode.com
- **Tagline**: "Smooth as butter"
- **Target**: Developers who want quick, polished tools

## Architecture

### Core Sections

| Section | Purpose |
|---------|---------|
| `/tools/` | CSS helpers, color converters, mini utilities |
| `/snippets/` | Ready-to-use code patterns |
| `/cheatsheets/` | Quick reference guides |
| `/ui/` | Component showcases |
| `/blog/` | Hoffbit articles, curated content |

### Design Philosophy

1. **Simple** - Each tool does one thing well
2. **Fast** - No bloat, quick loading
3. **Polished** - Professional, clean UI
4. **Mobile-friendly** - Works everywhere
5. **Accessible** - WCAG compliant

## Stack

- Next.js 14+ with App Router
- Tailwind CSS
- Solarized/Sandig color theme
- Self-hosted or Vercel

## Content Sources

- Original tools and utilities
- Hoffbit blog articles
- Curated external resources
- Community contributions

## Phase 1 Goals

1. Set up basic Next.js project structure
2. Create landing page
3. Add 2-3 initial mini tools
4. Set up blog section
5. Deploy to production

## Notes

- Focus on quality over quantity
- Each tool should be useful standalone
- Documentation is important
- SEO-friendly for discoverability
